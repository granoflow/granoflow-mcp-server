# Legacy Task Analysis Workflow

Use this reference only when resolving or reading a historical Task Analysis or
its managed summary. It is not the new-task Analysis owner.

For every new task and material amendment, use
`task-work-document-workflow.md` with `task-work-document-template.md`.

Legacy behavior remains readable:

- preserve the immutable Analysis attachment and App-owned hash;
- preserve `granoflow-analysis-summary` markers and all user text outside them;
- do not infer that finalized Analysis authorized Planning or Execution;
- read legacy learning/software Profiles, Skill evidence, Card Checkpoint, and
  user confirmations when they remain relevant;
- never claim legacy content was migrated.

If a material change is needed, create a self-contained `task_work` working
copy, apply current optional-section triggers, confirm Analysis, decide whether
Planning is required, upload an immutable version, and switch the active pointer
only after content/SHA-256 readback.

The old localized triggers such as `请分析第一个任务` now route to the Task Work
Document workflow for new work.
